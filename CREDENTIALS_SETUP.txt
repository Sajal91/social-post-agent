# SocialPostAgent — Credentials Setup Guide
# Copy .env.example to .env and fill in each value below.

================================================================================
DATABASE
================================================================================

MONGODB_URI
  Used in: server/src/config/db.ts
  Purpose:  MongoDB connection string for workflow sessions and run history
  How to get:
    - Local:  mongodb://127.0.0.1:27017/socialpostagent
    - Cloud:  Create free cluster at https://www.mongodb.com/cloud/atlas
              Copy connection string and replace password

================================================================================
GOOGLE GEMINI (AI text + images)
================================================================================

GEMINI_API_KEY
  Used in: server/src/services/gemini-content.ts
           server/src/services/gemini-image.ts
  Purpose:  Generate topics, captions, revisions, and images
  How to get:
    1. Go to https://aistudio.google.com/apikey
    2. Create an API key
    3. Paste into .env

================================================================================
WHATSAPP CLOUD API (Meta)
================================================================================

WHATSAPP_VERIFY_TOKEN
  Used in: server/src/routes/whatsapp-webhook.ts
  Purpose:  Secret string you choose; Meta sends it back during webhook verification
  How to get:  Pick any random string (e.g. my_secure_verify_token_123)

WHATSAPP_ACCESS_TOKEN
  Used in: server/src/services/whatsapp.ts
  Purpose:  Send messages, upload media, interactive lists/buttons
  How to get:
    1. https://developers.facebook.com/apps/
    2. Create app → Add WhatsApp product
    3. API Setup → copy temporary or permanent access token

WHATSAPP_PHONE_NUMBER_ID
  Used in: server/src/services/whatsapp.ts
  Purpose:  Your WhatsApp Business phone number ID
  How to get:  Meta Developer App → WhatsApp → API Setup → Phone number ID

WHATSAPP_BUSINESS_ACCOUNT_ID (WABA ID)
  Used in:  Setup only (not in code, but required for webhook subscription)
  Purpose:  Link your WhatsApp Business Account to your app
  How to get:  Meta Developer App → WhatsApp → API Setup → WhatsApp Business Account ID
  IMPORTANT — Run once after setup:
    POST https://graph.facebook.com/v21.0/{WABA_ID}/subscribed_apps
    Header: Authorization: Bearer {WHATSAPP_ACCESS_TOKEN}
    Without this, inbound messages will NOT reach your webhook.

PUBLIC_BASE_URL
  Used in: server/src/services/gemini-image.ts (image URLs for Instagram/Facebook posting)
           server/src/index.ts (serves /uploads)
  Purpose:  Public HTTPS URL of your server (required for image posting to social platforms)
  How to get:
    - Dev:  Run ngrok http 5000 → use https URL (e.g. https://abc123.ngrok.io)
    - Prod: Your deployed server URL

================================================================================
FACEBOOK PAGE POSTING
================================================================================

FACEBOOK_PAGE_ID
  Used in: server/src/services/meta-facebook.ts
  Purpose:  Target Facebook Page for posts
  How to get:
    1. Create a Facebook Page
    2. Graph API Explorer → GET /me/accounts → copy page id

FACEBOOK_PAGE_ACCESS_TOKEN
  Used in: server/src/services/meta-facebook.ts
           server/src/services/meta-instagram.ts
  Purpose:  Post to Facebook Page and Instagram (same token)
  How to get:
    1. Graph API Explorer with pages_manage_posts permission
    2. GET /me/accounts → copy page access_token
  Required permissions: pages_manage_posts, pages_read_engagement

================================================================================
INSTAGRAM POSTING
================================================================================

INSTAGRAM_BUSINESS_ACCOUNT_ID
  Used in: server/src/services/meta-instagram.ts
  Purpose:  Instagram Business/Creator account user ID
  How to get:
    1. Convert Instagram to Business/Creator account
    2. Link Instagram to your Facebook Page (Instagram settings)
    3. GET /{PAGE_ID}?fields=instagram_business_account
    4. Copy instagram_business_account.id
  Required permission: instagram_business_content_publish (Meta App Review)

Note: Instagram posting requires PUBLIC_BASE_URL so Meta can fetch the image URL.

================================================================================
LINKEDIN POSTING
================================================================================

LINKEDIN_ACCESS_TOKEN
  Used in: server/src/services/linkedin.ts
  Purpose:  OAuth access token for posting
  How to get:
    1. https://www.linkedin.com/developers/apps
    2. Create app → Auth → generate token (or OAuth flow)
    3. Requires Community Management API product (partner approval may apply)

LINKEDIN_ORGANIZATION_URN
  Used in: server/src/services/linkedin.ts
  Purpose:  Target company page (format: urn:li:organization:12345678)
  How to get:
    1. LinkedIn Developer Portal → your app
    2. Use Organization lookup API or copy from company page admin tools

Note: If LinkedIn is not configured, the bot skips LinkedIn and continues with other platforms.

================================================================================
OPTIONAL / DEVELOPMENT
================================================================================

CLIENT_URL
  Used in: server/src/index.ts (CORS)
  Default:  http://localhost:5173
  Purpose:  Allow React dashboard to call API in development

POSTING_DRY_RUN
  Used in: server/src/services/meta-facebook.ts, meta-instagram.ts, linkedin.ts
  Values:   true | false (default false)
  Purpose:  Skip real social API calls; still records mock success in dashboard

PORT
  Used in: server/src/index.ts
  Default:  5000

META_APP_ID / META_APP_SECRET
  Not read directly by code but needed for Meta OAuth and long-lived tokens
  How to get: Meta Developer App → Settings → Basic

================================================================================
WEBHOOK SETUP CHECKLIST
================================================================================

1. Start server: npm run dev:server
2. Expose with ngrok: ngrok http 5000
3. Set PUBLIC_BASE_URL in .env to ngrok HTTPS URL
4. Meta Developer App → WhatsApp → Configuration:
   - Callback URL: {PUBLIC_BASE_URL}/webhook/whatsapp
   - Verify token: same as WHATSAPP_VERIFY_TOKEN in .env
   - Subscribe to: messages
5. Subscribe WABA to app (see WHATSAPP_BUSINESS_ACCOUNT_ID above)
6. Add your phone as test recipient in Meta dashboard
7. Send WhatsApp message: "Hey, Generate content"
